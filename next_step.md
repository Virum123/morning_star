# Morning Star - Next Steps (Mobile & Web App Migration)

데스크톱 앱(PC 실행파일)으로 완성된 Morning Star를 **구글 플레이스토어**나 **토스(Toss) 앱 내 웹 서비스**로 확장하기 위한 기술적 다음 단계(Next Steps)입니다.

## 1. 구글 플레이스토어(Android) 출시를 위한 변환

가장 추천하는 방법은 기존에 만들어둔 React UI 코드를 재사용하여 안드로이드 앱으로 패키징하는 것입니다.

### A. 기술 스택 변경 (택 1)
*   **Capacitor (가장 쉬움):** 현재의 React 코드(`ui` 폴더)를 그대로 모바일 웹뷰 형태의 네이티브 앱(`.apk` / `.aab`)으로 감싸주는 프레임워크입니다.
*   **React Native (성능 중심):** 코드 구조는 비슷하나, UI 태그를 전부 모바일 네이티브 대응 태그(`<View>`, `<Text>`)로 재작성하고 컴파일하는 방법입니다.

### B. 필수 로직 교체
*   **파일 시스템(파이썬) 한계:** 현재의 파이썬 기반 로컬 `app_data` 폴더 파일 입출력을 사용할 수 없습니다.
*   **브라우저 스토리지 도입:** IndexedDB, LocalStorage 혹은 브라우저 파일 시스템 API를 활용해 자바스크립트 기반으로 마크다운 파일을 기기 내부에 저장해야 합니다. 백엔드(app.py) 없이 100% 프론트엔드로 동작하게 만듭니다.

### C. 플레이스토어 등록 절차
1.  **개발자 계정:** 구글 플레이 콘솔 가입 및 등록비($25, 평생 1회) 결제
2.  **앱 빌드:** Android Studio 등을 구동해 `.aab` 릴리즈 빌드 추출
3.  **정보 등록:** 앱 아이콘, 스크린샷, 텍스트, 개인정보처리방침 기입
4.  **심사 및 출시:** 첫 출시는 보통 3~7일 가량 소요.

## 2. 토스(Toss) 앱 등 웹 서비스로의 오픈

앱 설치 없이 링크만으로 접근하거나 특정 슈퍼앱 메뉴 안에 넣고 싶을 때 사용하는 방법입니다.

### A. 기술 스택 (모바일 웹/PWA)
*   현재 사용 중인 Vite + React 코드를 인터넷 망(Vercel, AWS 등)에 올려 웹앱 형태로 호스팅합니다.
*   반응형 웹 디자인을 강화하여, 모바일 브라우저에서도 버튼이 큼직하게 잘 눌리도록 수정해야 합니다.

### B. 토스 앱 연동(B2B) 구조
*   단순히 외부 링크 아웃링크 방식으로 토스 채팅방 등에 공유하는 것은 웹 호스팅 리소스 링크만 있으면 가능합니다.
*   하지만 **토스 앱 내 정식 혜택 탭이나 미니 서비스**로 입점하려면 오픈 API 수준이 아니라 비바리퍼블리카 측과의 **정식 제휴 파트너십/사업 제안**이 선행되어야 합니다.

## 요약 프로세스

1.  **UI 모바일 뷰 최적화:** 브라우저 창을 세로(스마트폰 너비)로 줄여도 깨지지 않게 CSS 미디어 쿼리를 다듬습니다.
2.  **클라이언트-온리(Client-Only) 전환:** 파이썬(pywebview) 의존성을 버리고 브라우저 스토리지만 쓰도록 `api.js`를 교체합니다.
3.  **Capacitor 연동:** `@capacitor/core` 및 `@capacitor/android`를 붙여서 스마트폰 테스트 빌드를 뽑습니다.
